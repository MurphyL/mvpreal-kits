# 项目概述

- 技术栈：fyne/v2


## 目录结构入口
- `./` - 为源代码目录
- `gui/` - 为界面代码目录
- `feat/` - 为 Tab 功能（Feature）模块代码目录

## 命名规范：
- 测试文件：`<原文件名>_test.go`（与被测试文件同目录）
- 测试函数：`Test_<函数名>_<场景>`
- 包名：小写、简短、有语义：model、service、utils，避免缩写（除非广泛认知）
- 函数 / 方法 / 变量：驼峰命名法，公共首字母大写，私有小写
- 常量：全大写，驼峰命名法，不使用全大写下划线

## 常用命令（必须准确）

- 安装依赖：`go get -u all`
- 项目构建：`go build -o mvpreal`
- 依赖整理：`go mod tidy`

## 约束（团队共识）

- 默认先用 Plan Mode 分析，再动代码
- 任何会影响行为的改动：必须补测试/补文档
- 涉及凭证/权限：先安全审查，再合并
